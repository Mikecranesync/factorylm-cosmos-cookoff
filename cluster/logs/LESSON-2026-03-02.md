## AI Mistakes This Session

1. **Python 3.9 compatibility miss**
   - Used `List[FaultDiagnosis] | None` syntax in `diagnosis/prompts.py`
   - Python 3.9 doesn't support `X | None` union syntax — needs `Optional[X]`
   - **Fix:** Changed to `Optional[List[FaultDiagnosis]]`
   - **Root cause:** Assumed Python 3.10+ was available. CHARLIE runs macOS system Python 3.9.6.
   - **Rule candidate:** Always use `Optional[X]` or `from __future__ import annotations` for type hints in this repo.

2. **Frame center initialization tied to calibration flag**
   - `_center_x` and `_center_y` only set when `_calibrated == False`
   - `set_baseline_rpm()` sets `_calibrated = True` before any frame is processed → center stays at (0, 0) → everything reads as MISTRACK
   - **Fix:** Separated `_center_initialized` flag from `_calibrated` flag. Center is set on first frame regardless of calibration state.
   - **Root cause:** Conflated two independent concerns (geometry vs. speed baseline) into one flag.

3. **Debounce initial value blocked first crossing**
   - `_last_crossing_time` initialized to `0.0`
   - First crossing at timestamp 0.005: `0.005 - 0.0 = 0.005 < debounce(0.01)` → rejected
   - **Fix:** Initialize `_last_crossing_time = -999.0` so first real crossing always passes.
   - **Root cause:** Used `0.0` as "no crossing yet" sentinel, but `0.0` is a valid timestamp in test scenarios.

4. **Test assertions too tight**
   - `assert offset_px > 50` when actual offset was exactly 50 (tape at x=50, center at x=0 due to bug #2)
   - **Fix:** Used more extreme test values (tape at x=20) and compared against the actual threshold constant.

## Fine-Tuning Candidates

1. **Pattern: Python version detection at import time**
   - This repo runs on Python 3.9.6 (macOS system). Every file should either use `from __future__ import annotations` or stick to `Optional`/`Union` syntax.
   - Good training example: the before/after of the `List[X] | None` → `Optional[List[X]]` fix.

2. **Pattern: Separate initialization concerns**
   - When a class has multiple init phases (geometry setup vs. calibration vs. first data), track each independently.
   - The `_center_initialized` / `_calibrated` split is a clean training example.

3. **Pattern: Sentinel values for "never happened" timestamps**
   - Use `-float('inf')` or a large negative number, not `0.0`, for "no event yet" timestamps.
   - `0.0` is a valid epoch time and a common test timestamp.
